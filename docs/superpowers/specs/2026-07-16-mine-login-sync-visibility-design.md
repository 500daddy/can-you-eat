# “我的”页登录入口与同步提示调整

## 目标

- 未登录时不显示默认头像和头像边框。
- 账号卡中只保留一个登录操作，避免“登录”和“去登录”重复。
- 不再长期向用户展示无法解决的“未同步/重试”状态。

## 已确认方案：A1

### 未登录状态

- 账号主行不渲染头像节点，文字从卡片左侧对齐。
- 主行显示“微信登录”、辅助文案和唯一操作“登录 ›”，整行可点击。
- 家庭组行保留“创建家庭组”和价值说明，右侧显示非按钮状态“登录后可用”。
- 家庭组行未登录时不响应点击，不再重复跳转登录页。

### 已登录状态

- 恢复展示用户头像、昵称和“设置 ›”。
- 家庭组行变为可点击的家庭摘要，进入家庭共享管理页。

## 同步状态策略

- 保留账号服务的后台同步、失败分类和本机待同步数据，不删除数据保护逻辑。
- 删除“我的”页上的未同步黄色状态条与重试入口。
- 删除账号设置页长期展示的“本机食材还未同步完成”卡片。
- 登录、保存账号或食材保存本身失败时，仍立即显示明确的失败提示。
- 后台同步失败不删除本机记录，后续进入页面时可继续自动尝试。

## 实现边界

- 仅调整“我的”页和账号设置页的展示与手动重试交互。
- 不改动账号登录、本地数据保留、家庭共享和云端同步的底层数据结构。
- 不新增图片资源或依赖，不增加小程序包体积。

## 验收标准

- 未登录截图中不存在头像框，只有一个“登录”动作。
- “登录后可用”不可点击，登录后家庭摘要可正常进入管理页。
- “我的”页和账号设置页都不再出现长期未同步状态和手动重试按钮。
- 已登录头像、账号设置、家庭共享和后台同步测试保持通过。
