# 宝宝食材小管家 UI Style Guide

> 当前版本用于 UI 定稿和小程序落地前的视觉约束。推荐评审原型：`design-demos/宝宝食材小管家-ui-prototype-v2.html`。v1 原型保留为早期方向对照。

## 设计方向

整体方向是“养娃新手村 / 食材小管家”：奶油纸底、森林绿主色、深棕像素描边、木牌标题、纸质卡片、轻游戏化状态徽章。v2 比 v1 更接近目标图，优先作为后续小程序开发的 UI 来源。

设计优先级：

1. 状态判断清晰，用户能快速知道“宝宝是否建议食用”。
2. 像素风负责亲和感，不承载关键判断。
3. 所有文案保持保守，不使用“绝对安全”“放心吃”“一定能吃”等承诺。

状态色规则：

- `baby_ok`：绿色标签，底色 `#eaf7d8`，描边 `#69aa63`。
- `baby_today`：黄绿或橙色标签，强调“今天优先”。
- `adult_only`：橙色标签，强调“成人参考，不是安全承诺”。
- `not_recommended` / `expired`：红色标签和浅红底，但避免恐吓式大面积红色。
- `finished`：灰色标签，降低视觉优先级。

## 字体与排版

- 小程序正文使用系统字体：`-apple-system`, `PingFang SC`, `Microsoft YaHei`, sans-serif。
- 大标题加粗，使用 900 字重，允许轻微描边或阴影形成像素标题感。
- 手机内字号建议：
  - 页面标题：17-20px
  - 卡片标题：14-16px
  - 正文说明：11-13px
  - 状态标签：10-12px
  - Tab 文案：10px
- 行高保持 1.45-1.7，详情页说明文案不低于 1.6。

## 组件规范

### 主按钮

- 高度：40-44px。
- 背景：绿色渐变或纯 `#3f8a43`。
- 描边：2-3px 深绿。
- 阴影：底部硬阴影，模拟像素按钮按压感。
- 文案：动词优先，如“添加食材”“拍照识别”“记录食用”。

### 次按钮

- 用于拍照识别、编辑、手动搜索。
- 背景：奶油黄 / 木牌黄。
- 描边：金棕色。
- 不与主按钮抢绿色优先级。

### 食材卡片

结构固定：

1. 左侧 48px 食材 icon。
2. 中间食材名、状态标签、保存方式、购买日期。
3. 右侧箭头或操作按钮。

卡片边框根据状态变化：

- 建议宝宝吃：绿色描边。
- 可给大人吃：橙色描边。
- 不建议宝宝食用：红色描边。
- 卡片圆角建议 10-14px；保留像素描边，但避免 4-6px 的尖锐直角。

### 状态标签

必须使用“建议 / 参考 / 判断”类表达：

- 建议给宝宝吃
- 建议今天给宝宝吃
- 可留给大人吃
- 不建议给宝宝食用
- 不建议继续食用
- 已处理

标签圆角建议 8px 左右，和卡片保持同一套圆润像素感。

禁止文案：

- 绝对安全
- 一定能吃
- 放心吃
- 不会变质

### 表单

- 输入框高度 36-40px。
- 圆角 8-10px。
- 必填字段用小红点或“*”，不要大面积警示色。
- 分段控件用于保存方式：常温 / 冷藏 / 冷冻。
- 默认选择推荐保存方式，减少用户填写成本。

### 空状态

空状态必须包含：

- 一个大号像素吉祥物或冰箱占位。
- 标题：“还没有记录宝宝食材”。
- 说明：“添加第一个食材，看看什么时候适合给宝宝吃吧～”。
- 主按钮：“添加食材”。
- 次按钮：“拍照识别”。

## 像素资产规范

原型阶段优先使用内联 SVG / CSS 像素 sprite 占位，避免直接使用大 emoji 作为主视觉。正式资产建议用 Imagine2 生成后人工清理。

推荐规格：

| 类型 | 设计尺寸 | 导出 |
| --- | --- | --- |
| 食材小 icon | `32x32` | `@2x` / `@3x` PNG |
| 食材重点 icon | `48x48` | `@2x` / `@3x` PNG |
| 宝宝头像 / 吉祥物 | `96x96` | `@2x` / `@3x` PNG |
| 顶部村庄 banner | 高度 120-160px | PNG/WebP |
| 状态小徽章 | `24x24` 或 `32x32` | PNG |

统一要求：

- 透明底 PNG。
- 像素网格统一，优先 32px 基准。
- 描边使用深棕色，不使用纯黑。
- 色板控制在 12-18 个主色以内。
- 食材 icon 需要一眼可识别，不能为了可爱牺牲辨识度。

## 小程序落地建议

- 在 `app.wxss` 定义全局色彩变量；微信小程序可使用 CSS 变量时优先复用。
- 建议拆分公共组件：
  - `components/status-tag`
  - `components/food-card`
  - `components/pixel-empty`
  - `components/storage-segmented`
  - `components/pixel-button`
- 首页、提醒中心、详情页复用同一套状态映射，避免文案和颜色漂移。
- 正式开发时不要把像素装饰做成大图背景铺满页面，避免压低可读性和性能。

## 验收清单

- 首页 3 秒内能看出优先给宝宝吃的食材。
- 成人参考状态不被误读为“宝宝还能吃”。
- 不建议食用状态有足够警示，但不过度吓人。
- 像素 IP 只增强亲和感，不影响信息层级。
- 所有页面底部或详情页包含保守免责声明。
